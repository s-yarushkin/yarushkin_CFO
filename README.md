## Исправление v0.8.2

Исправлен баг, когда после сохранения записи таблица могла не обновляться из-за кеширования API/GET-запросов Next.js.

Что изменено:
- для API routes добавлено `dynamic = "force-dynamic"` и `revalidate = 0`;
- frontend fetch использует `cache: "no-store"`;
- добавлена кнопка `Обновить`;
- после сохранения записи таблица повторно подтягивает модель из Supabase;
- добавлено отображение времени последнего обновления.
